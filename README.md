# ПОЯСНИТЕЛЬНАЯ ЗАПИСКА ПО ПРОЕКТУ "ПРИЛОЖЕНИЕ ДЛЯ УЧЕТА ЗАПИСЕЙ"

В этой пояснительной записке будет описано, какие технологии, модули и язык программирования я использовал, как разработчик, в данном проекте

#### Использованные модули:
- **_json_** - Сериализация задач в **_JSON_** формат и обратно. Превращает словари/списки в текст для сохранения в файл `tasks.json` и наоборот.
- _**os**_ - Проверка существования файла `tasks.json` перед загрузкой. Нужен, чтобы не пытаться открыть несуществующий файл.
- _**datetime**_ - Работа с датами: получение текущей даты для created_at, сравнение deadline с текущей датой для определения просрочки, валидация формата введенной даты.
- _**csv**_ - Экспорт списка задач в CSV файл. Позволяет открыть отчет в Excel для дальнейшего анализа.

#### Системные требования

- **Python 3.6** или выше
- Операционная система: **Windows** / **Linux** / **macOS**
- Дополнительные библиотеки не требуются, т.к. все нужное библиотеки и модули, встроенные впроект, уже имеются в Python

#### Как запустить и начать работу

Для запуска программы можно воспользоваться у вас должен быть установлен python той версии, которая указана в системных требованиях, иначе не запустится. Далее вы можете запустить `main.py` через pycharm, либо через vs code, либо через командную строку с помощью команды `python main.py`.

#### Система пользователей

Также в данном проекте я решил реализовать простую систему пользователей. В моей программе есть всего 2 типа пользователей: администратор либо обычный пользователь. Чтобы войти в режим администратора, нужно в самом начале программы ввести имя пользователя "admin", а чтобы войти в режим обычного пользователя, вводим любое другое имя.

Пример реализации:

```bash
==================================================
  УЧЕТ ЗАДАЧ - Строительная фирма
==================================================

Ваше имя (или 'admin'): admin

Режим: АДМИНИСТРАТОР (все задачи)

КОМАНДЫ:
  add                           - Добавить задачу
  list                          - Показать все задачи
  show [новая|в работе|выполнена] - Задачи по статусу
  update [id]                   - Изменить задачу
  done [id]                     - Отметить как выполненную
  delete [id]                   - Удалить задачу
  responsible [имя]             - Задачи сотрудника
  overdue                       - Просроченные задачи
  export                        - Экспорт в CSV
  help                          - Справка
  exit                          - Выход

```

Реализация входа через обычного пользователя:

```bash
==================================================
  УЧЕТ ЗАДАЧ - Строительная фирма
==================================================

Ваше имя (или 'admin'): vlad

Режим: vlad (только свои задачи)

КОМАНДЫ:
  add                           - Добавить задачу
  list                          - Показать все задачи
  show [новая|в работе|выполнена] - Задачи по статусу
  update [id]                   - Изменить задачу
  done [id]                     - Отметить как выполненную
  delete [id]                   - Удалить задачу
  responsible [имя]             - Задачи сотрудника
  overdue                       - Просроченные задачи
  export                        - Экспорт в CSV
  help                          - Справка
  exit                          - Выход

```
